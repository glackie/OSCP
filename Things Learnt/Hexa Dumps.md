Note to myself on how to deal with hexa dumps



At the end there i have pasted and exxampled of what hexa dumps look like

From what i understand it is important to understand the first 3 bytes to see what type of file we are dealing here and then accordingly we can extract the file in that format to deal with it


We can also use [[CyberChef]] to deal with this

statements from writeup below:


Looking at the first 3 bytes, I’ll see `BZh`, the [magic bytes](https://en.wikipedia.org/wiki/List_of_file_signatures) for a .bz2 file.

I’ll convert back to binary with `xxd` and `-r` for reverse:

```
root@kali# cat password_backup_orig | xxd -r > password_backup.bz2
root@kali# file password_backup.bz2 
password_backup.bz2: bzip2 compressed data, block size = 900k
```
